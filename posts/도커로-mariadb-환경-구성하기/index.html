이미지 다운로드#

버전을 명시하지 않으면 latest 버전이 설치된다.

1

$ docker pull mariadb:${VERSION}

컨테이너 생성#

컨테이너 내부에 저장된 데이터들은 컨테이너가 삭제된다면 같이 삭제된다. 즉 stateful 하기 때문에 데이터의 영속성을 보장하는 stateless 한 설계를 위해 Docker 는 volumes 과 bind mounts 라는 두가지 옵션을 제공한다.

volumes 와 bind mounts 의 차이

1. Use volumes#

존재하지 않는 볼륨명 입력 시 입력된 볼륨명으로 볼륨이 생성되고, 생략 시 해시 값으로 볼륨이 생성된다.

MYSQL_ROOT_PASSWORD 는 필수 환경 변수 이며, 컨테이너 실행 실패 시 docker logs ${CONTAINER_NAME} 명령어로 내부 로그를 확인할 수 있다.

1
2

$ docker run -d -p 3306:3306 -e MYSQL_ROOT_PASSWORD=${MYSQL_ROOT_PASSWORD}
-v ${VOLUME_NAME}:/var/lib/mysql --name ${CONTAINER_NAME} mariadb

2. Use bind mounts#

1
2

$ docker run -d -p 3306:3306 -e MYSQL_ROOT_PASSWORD=${MYSQL_ROOT_PASSWORD} \
-v ${HOST}:/var/lib/mysql --name ${CONTAINER_NAME} mariadb

컨테이너 접속#

1

$ docker exec -it ${CONTAINER_NAME} bash
